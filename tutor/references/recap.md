# Session recap (subskill of tutor)

Load this reference (via `read`) when a session is wrapping up — either the learner says they're done, you've covered the agreed goal, or the conversation is naturally ending. The recap is a written artifact the learner keeps; it is where short-term understanding becomes long-term retention.

## When to load

- The learner signals the session is ending ("that's all for now", "gotta go", "thanks").
- The agreed learning goal has been achieved and checked.
- A natural stopping point after a substantial concept landed.

Do not load this mid-session or for trivial exchanges — a recap of a five-minute clarification is noise.

## What the recap must contain

Produce a single markdown block the learner can copy and keep. Use exactly these sections:

```
## Session recap — <date or topic>

### Goal
<the one-sentence learning goal agreed at the start; if it shifted, note the shift>

### What we covered
- <concept 1> — one line on the key idea, in the learner's own framing where possible.
- <concept 2> — ...

### Key takeaways
- <the 2-4 ideas that matter most, phrased so they're re-readable later without context>

### Still shaky / to revisit
- <concept or skill that didn't fully land; be honest, don't inflate>
- <specific misconception that recurred — name it so they can watch for it>

### Suggested next steps
- <one or two concrete things to practice before next session, tied to a toy domain — point at practice-problems if relevant>
- <the next concept that builds on this one, so the learning thread has a direction>
```

## Rules

- **Honest, not flattering.** If something is still shaky, say so. A recap that claims mastery the learner doesn't have sets up future frustration.
- **In the learner's words where you can.** Capture concepts as they phrased them during check-understanding, not as you originally stated them. Their phrasing is what they'll re-recognize later.
- **Forward-looking.** The "suggested next steps" section is what makes a recap a thread, not a tombstone. It should give a clear direction and, where useful, point at a toy practice problem or the next concept.
- **Concise.** This is a memory aid, not a textbook. A recap longer than the session is a failure mode. Cap each bullet at one line.
- **No solution code.** The recap may reference concepts and approaches, but never paste the learner's solution or any solution. It's a summary of understanding, not an answer key.

## Delivery

Present the recap block and say plainly: "Here's your recap — copy it somewhere you'll find it next session." Offer to adjust framing if any takeaway doesn't match their mental model. Do not assume they'll remember it without writing it down.
